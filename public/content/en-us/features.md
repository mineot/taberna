# Features

## Content that does not get trapped in code

Taberna keeps presentation and content in separate places. This organization makes information easier to find, review, and replace without navigating through interface components.

### Markdown for writing

Pages and section copy can be created in Markdown. Headings, lists, links, emphasis, and other familiar elements are available in a simple, readable format.

### JSON for organizing

Configuration files define the site title, description, menu, sections, images, and footer. Each language has its own complete and independent configuration.

## A flexible presentation

Sections can combine titles, subtitles, images, plain text, or Markdown content. You can also reverse the layout, adjust vertical alignment, emphasize a block with a different background, and bring multiple pieces of content together in a carousel.

## Designed for different screens

The layout adapts to phones and desktop computers. On smaller screens, navigation moves into a sidebar with focus management, keyboard controls, and scroll locking while it is open.

## Built-in language support

Visitors can choose from the available languages. Taberna also considers browser preferences and remembers the selection for future visits.

## Safer dynamic content

HTML generated from Markdown files is sanitized before it appears on the page. External menu links are also limited to valid HTTP and HTTPS addresses.

## A customizable appearance

Colors, contrast, and transition durations are centralized in semantic properties. This makes it possible to change the visual identity consistently without hunting for individual colors in every component.

## Ready for static publishing

Once configured and built for production, the result can be hosted as a static site on compatible services. See the [project documentation](https://github.com/mineot/taberna/blob/master/README.md) for installation details and available commands.
